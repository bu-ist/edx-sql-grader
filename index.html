Approach

For the SQL queries, we will not be developing any “front end” customizations to the edX platform; we will be relying solely upon the platform to provide an interface for managing questions and answers, and for handling all student interactions with the questions and answers.

Students are presented with a question and provided a text area to enter their answer in the form of a SQL query.

When the student clicks or taps the button to submit his or her answer, edX submits the response to the XQueue interface, which queues the response until our custom grader processes it.

A canonical “answer” query for the question will also be submitted along with the student response. The canonical query is part of the question definition and is managed using the edX LMS, i.e. the course instructors enter the canonical query along with the question.

When the grader processes a student response, it compares the result set of the canonical query to the result set of the query that the student submitted, i.e. it compares the result of the query that is known to generate the desired result to the result of the query that the student thinks will generate the desired result.

Initially, the grader will perform a simple, binary comparison of the two results. The student’s query will either generate a correct result, or it will not. Additional analysis may be added to the grader if time permits.

The grader then generates a response to be sent back to the edX platform. The response will contain:

  1. The “grade”, from 0 to 1. Initially, this will be either a 0 (result did not match expected) or a 1 (result did match expected)
  2. A HTML table representing the output of the student’s query
  3. A HTML table representing the desired output
